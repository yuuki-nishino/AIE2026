## 詳細内容

### AIエージェントとは何か

AIエージェントは「計画をたてて・ツールを使い・自動で試行錯誤する」システム。読み込むだけでなく、**操作系の処理**（ファイル作成、APIコール、ブラウザ操作等）を自律的に実行できる。

### 実際のユースケース事例

**1. KDDIの本部長AIエージェント「A-BOSS」**  
法人営業向けの提案内容を本部長相当の視点でレビューしてくれるAIエージェント。社内業務の意思決定支援に活用。

**2. パワポ作るマン**  
Web検索 → スライド作成 → 共有・エクスポートまでを自動化するエージェント。資料作成業務を大幅に効率化。

**3. M365操作代行！秘書エージェント**  
Microsoft 365（Outlook、Teams、Confluenceなど）を操作し、会議の手配や資料確認などを自律的に行うエージェント。AWSで動作する構成。

### AIエージェントの実装

**SDK: Strands Agents（推奨）**

```python
from strands import Agent

agent = Agent()
agent("AIエージェントって何？")
```

わずか3行でエージェントが動く。他にもLangChain、Mastra、ADK等のフレームワークが使えるが、いずれもシンプルなコードで書けることが共通点。

**ツール追加: @toolデコレータ**

```python
from strands import Agent, tool

@tool
def add(x, y):
    return x + y

agent = Agent(
    model="jp.anthropic.claude-sonnet-4-6",
    tools=[add]
)
```

関数を書いて`@tool`でデコるだけでエージェントにツールを追加できる。

### デプロイアーキテクチャ: Amazon Bedrock AgentCore

**エージェントのデプロイ先が従来の課題だった：**
- ストリーミングレスポンスに対応したい
- 時間のかかる非同期処理もある
- 認証認可や監視も必要
- なるべくコストをかけたくない、サーバレスにしたい

**解決策: AgentCoreランタイム**  
作ったエージェントをコンテナにしてデプロイするだけで、上記の課題をすべて解決できる。

```python
from strands import Agent
from bedrock_agentcore.runtime import BedRockAgentCoreApp

agent = Agent()
app = BedRockAgentCoreApp()

@app.entrypoint
async def run_agent(payload, context):
    prompt = payload.get('prompt')
    return {"result": agent(prompt).message}

app.run()
```

**フルサーバレスの鉄板アーキテクチャ：**
- **フロントエンド**: React + Amplify Gen2（GitHubにプッシュするだけでCI/CDが自動で動く）
- **バックエンド**: AgentCore Runtime（コンテナ）+ Cognito認証
- **IaC**: AWS CDK または Terraform でインフラ全体をコード管理

**動作履歴のトレース**: CloudWatchで自動的にエージェントの動作をトレース可能。監視SaaSを入れなくてもCloudWatchで十分な可視性が得られる。

### セキュリティ・権限管理

**AgentCore Identity機能**  
- OAuthクライアントを作成
- AgentCore SDKで認証が必要な関数を追加するだけで、SaaS連携時の認証処理を自動化

**AgentCoreポリシー**  
ツールの挙動をエージェントループの外から統制できる。これにより「特定のツールを操作させたくない」という要件を、ツール内のコードではなく外部ポリシーとして管理できる。

### 品質改善・評価

**AgentCore評価機能**  
- 選んだメトリクスでエージェントの行動を自動採点
- 低い点数のログだけを見て改善に集中できる

**AgentCore最適化**  
- システムプロンプトやツール説明の改善案を自動生成
- A/Bテストも可能

### 社内エージェントの管理

**AWSエージェントレジストリ**  
社内のエージェントやツールが増えすぎて管理しきれなくなった場合に活用。登録・承認のフローを設けることで、組織的なガバナンスを保ちながらエージェントの普及を図れる。

### AgentCore Payments（新機能）

ステーブルコインを使って、有料コンテンツへの少額決済をエージェントが自動管理できる機能。エージェントが必要なAPIや情報を自律的に課金して利用する未来を見据えた機能。

### ユースケースを見つけることの重要性

> 「今、一番足りないのはユースケース」

AIエージェントがコーディング以外の業務に活かせるはずだが、まだまだ利用されていない。

**みのるん自身の実践**
- 日常業務をすべてClaude Codeで爆速化（経費精算、勤怠登録、稟議申請などの雑務から会議の文字起こし・アジェンダ作成まで）
- 個人の家計管理や海外旅行の手配なども丸投げ
- 自分がAIで自動化できた業務をWebアプリにして非エンジニアも使えるようにする

**Vibe商談**  
商談中にデモアプリをこっそり作り、商談の最後に「もしかしてこれですか？」と見せる手法。そこからお客様の真の要件が出てくることが多い。

### 越境するマインドの重要性

見えていないユースケースを提案できるエンジニア、自分の役職だけでなく越境するマインドを持つ人材が重宝される時代になる。

## 社内への示唆・活用ポイント

- **自社業務に特化したエージェントを作ることのROIが高い** — 既製品のAIツールは汎用的すぎて自社要件を満たせない場合が多い。エンジニアが業務部門と連携して自社専用エージェントを作ることに価値がある
- **まずIaCでテンプレートを作ることを優先する** — バックエンド・フロントエンド・インフラを1セットのテンプレートとして作っておくと、2本目以降のエージェントを素早く量産できる
- **AgentCore Runtimeは自社でサーバ管理するより現実的** — サーバレスでコストも低く、ストリーミングやトレースも標準で対応しているため、独自インフラを立てるより優先して検討すべき
- **「自分の業務をAIで置き換えてみる」から始める** — 複雑なユースケースを最初から狙わず、自分の日常業務を1つずつAI化していくことで、社内展開できるユースケースの種が自然と見つかる
- **評価・採点の仕組みを最初から組み込む** — エージェントをデプロイして終わりではなく、パフォーマンスを継続的に計測・改善するためのメトリクスを最初から定義しておく
